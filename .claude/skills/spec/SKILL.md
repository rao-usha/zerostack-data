## Key Rules

- NEVER skip test skeleton creation
- ALWAYS set `.active_spec` — without it, enforcement hooks will block edits
- Test file naming MUST match the `Test file:` field in the spec
- Rubric items are REQUIRED checks, not suggestions
- The spec is a living document — update it as implementation reveals new requirements
